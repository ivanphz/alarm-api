# 架构与设计总纲（ios-alarm-api / Smart Schedule Gateway）

> 这份是**总纲**：讲清楚整个系统怎么分层、为什么这么分、各模块职责、以及贯穿全局的
> 两大原则——**解耦**和**闹钟可靠性模型**。专项细节见文末《文档索引》。

---

## 1. 系统一句话

**网关(Cloudflare Worker)算"此刻家里各设备该是什么状态 + 该有哪些闹钟"，
手机端的一组「刺客」定时来问、然后照着执行。** 网关只出结论，不碰手机；手机只执行，不做判断。

```
   [日历/请假/上帝模式]  [外部项目:还款/签到...]  [workdays-core:节假日]
            │                     │                        │
            ▼                     ▼                        ▼
        ┌──────────────────── 网关 Worker ────────────────────┐
        │  规则引擎 → 设备状态 + 闹钟清单(JSON)                 │
        └───────────────────────────┬─────────────────────────┘
                                     ▼  HTTP GET(带 key)
        ┌──────────────── iPhone 刺客(快捷指令)────────────────┐
        │  DNDTick → ApplyFocus/Silent/Volume + SyncAlarms      │
        └──────────────────────────────────────────────────────┘
```

## 3. 贯穿原则一：解耦（改一处不塌全局）

系统有三层解耦，任何一层的坏输入/坏配置都不该波及其它。

### 3.1 设备字段之间解耦（订阅命名规则）
`focus` / `silent` / `media_volume` 各是独立字段，通过 `USE` **订阅命名规则**（如 `dnd`），
而非"字段跟字段"。silent 复用 dnd 纯属恰好合用，与 focus 无引用关系：
**删掉 focus 整节，silent 照常从 dnd 渲染。** 没人订阅的规则会被审计标为孤儿。详见 device-state.js。

### 3.2 外部输入之间解耦（各自隔离 + 最外层兜底）
每个外部输入都被单独的 `try/catch`（+ 超时）圈起来，一个坏了不影响其它：

| 外部输入 | 隔离方式 | 坏了会怎样 |
|---|---|---|
| 家庭日历 ICS | 逐条 URL try/catch | 该条跳过，其余日历照常 |
| 外部闹钟源(JSON/ICS) | 逐源 try/catch + 5s 超时 + 逐项拒收 | 该源跳过，日志记"拒/警"计数 |
| `env.EXTERNAL_ALARMS` | try/catch | 坏 JSON 整体忽略，公开源不受影响 |
| workdays-core 节假日 | try/catch | 降级为自然周末推演（调休失效，主流程继续） |
| **以上都没接住的意外** | **最外层兜底网** | **返回合法降级响应，state 全 null，绝不崩** |

**最外层兜底网**（index.js 主 handler）：任何未被下游接住的异常——内部逻辑边界、
没预料的输入组合、未来改代码引入的 bug——都在这里被接住，返回 HTTP 200 + 一个格式合法
但安全的降级响应（`current_state.state` 全 null → 手机 Apply\* 空转、不误建闹钟；
预建固定闹钟常驻不受影响）。错误进 `error`/`humanReadable` 便于排查。
**保证：手机端永远拿得到合法 JSON，绝不因网关异常而整条同步失效。**

### 3.3 手机状态与网关解耦
网关**从不**假设手机当前状态，只下发"应该是什么"。ApplyFocus 等在手机侧读当前 focus
自行决策（守卫、切换）。网关无状态、可随时重算，两侧靠"JSON 契约"通信。

## 4. 贯穿原则二：闹钟可靠性模型（决定 Fixed vs 动态）

这是全系统最容易踩错、也最关键的一条。iOS 有两个硬约束逼出了两类闹钟：

- **快捷指令 `Create Alarm` 建的闹钟，震动/铃声只能用系统默认**，指定不了。
- **快捷指令没有"改现有闹钟时间"的动作**，只能 建/开/关/删。

由此分出两类，判据只有一句：**这条闹钟需不需要自定义震动/铃声，或"绝不能漏响"？**

| | Fixed 预建 `Gate-Fixed-*` | 动态 `Gate-Class-*` / `Gate-ES-*` / `Gate-Dynamic-Event-*` |
|---|---|---|
| 谁建 | 你在手机上**手工预建**，配好震动/铃声 | 网关按清单**随建随删** |
| 网关做什么 | 只 **开/关** | 下发具体点，手机建/关对账 |
| **响几点由谁定** | **手机**（改代码 scheduledAt 无效，只是镜像+窗口判断） | **网关/上游**（时间编进 label，改时间→关旧建新） |
| 自定义震动/铃声 | ✅ 能 | ❌ 只能默认 |
| **可靠性（关键）** | **常驻**：过去 24h 内**任一次**同步成功设为 ON 即会响；某次同步失败顶多"该关没关"多响一次 | **最近一次**同步必须成功且清单含它，否则当次**漏响** |
| 失败方向 | 多响（闹了总比没闹强） | **漏响**（可能不响） |
| 适合 | **叫醒**（当天第一个起床铃）、需要特定震动/铃声 | 后续课程、外部提醒、时间会变、漏了不致命 |

**落地准则**：
- 每天**第一个把你从睡着叫醒的闹钟** → 必须 Fixed（不容漏 + 时间稳定）。
- 需要特定震动/铃声组合（如"先震后响"两条）→ Fixed。
- 时间会变（如上课随寒暑假）、用默认铃可接受、漏了不致命 → 动态。

### 4.x focus 的 OFF 语义 与 only_if_current（设计，勿忘）
`only_if_current` 字段就是为"OFF 时要不要挑 focus"而设，语义**故意**如此：
- 有 `only_if_current`（守卫值）→ OFF **只在当前 focus == 守卫值时才关**（保护手动开的 Sleep/Work）。
- 无 `only_if_current`（为空）→ OFF **不管当前是什么 focus，一律关掉当前 focus**（"到点清场"）。

为什么：若无守卫也"只关 Mode"，这个字段就没用了。字段存在的意义正是让网关**按需**决定要不要挑 focus。
（07:40 配 `"Do Not Disturb"`=只在 DND 时解除；13:29 不配=到点清掉当前任何 focus。）本地实现见 `PHONE.md` §3。

### 4.y 线上 ↔ 本地契约（防脱节）
本地会读 `state` 的这些字段并如此消费：`focus`(对象→ApplyFocus 开关切)、`silent`("ON"/"OFF"/null)、
`media_volume`(0~1/null)、`sync_alarms_flag`("yes"/"no"→是否跑 SyncAlarms)。改任一字段的含义/取值，
**必须同步改 `PHONE.md` 与本地快捷指令**，否则"网关发 A、手机认 B"。逐动作真相以 `PHONE.md` 为准。

**不要用 between/时间容差去"纠正"闹钟时间**：Fixed 的时间在手机（网关不下发时间去比）；
动态的时间不一致时**本就该重建**（关旧建新），容差匹配会和它打架，把"该重建"误判成"将就用旧的"，
绕回时间不更新的 bug。稳定靠 Fixed 常驻，精准靠动态重建，各取所需。

## 5. 动态闹钟命名与对账（手机端要认的）

- 命名族：`Gate-Fixed-*`（预建开关，**不进** sweep；含 `Gate-Fixed-Class-<id>` 上课固定闹钟）、
  `Gate-Class-*`/`Gate-ES-*`/`Gate-Dynamic-Event-*`（动态，进 sweep）。动态族时间编进 label 尾巴（`...-HHMM`）。
- **周末上课是两种形态的混合**（V11.2）：同课在"时段时间==锚时间"的日子走固定 `Gate-Fixed-Class-<id>`
  （可靠常驻），时间不同的日子自动降级动态 `Gate-Class-*`。详见 config.WEEKEND_CLASS 与 DEVLOG §5.1。
- 手机端 SyncAlarms 对账两步：① 清单里有→建/开；② 手机上**动态族**里不在清单的→关。
- **sweep 匹配条件（本地实际采用：显式多前缀 ANY）**：
  `名称 contains "Gate-Dynamic-Event" 或 "Gate-ES" 或 "Gate-Class"`。
  覆盖三个动态族、排除预建 `Gate-Fixed-*`。**刻意保持显式**（不写成 `Gate- 且 not Gate-Fixed`），
  加新动态族时手动加一条前缀——让维护者清楚知道在管哪些族。以 `docs/PHONE.md` §6 为准。

## 6. 失败降级速查

| 出事 | 系统行为 |
|---|---|
| 某条日历 URL 拉失败 | 该日历跳过，其余照常，日志告警 |
| 某个外部闹钟源超时/脏 | 该源跳过，日志记拒收计数，其余照常 |
| workdays-core 异常/空 | 退自然周末推演（调休失效），主流程出结果 |
| 任何未预料的内部异常 | 最外层兜底网：合法降级响应，state 全 null，不崩 |
| 某次手机同步没跑成功 | Fixed 闹钟常驻仍响；动态闹钟当次可能不生效 |
| 鉴权密钥缺失/错误 | 401 锁死（fail-closed，绝不静默裸奔） |
